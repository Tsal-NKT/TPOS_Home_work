DockerFile - это инструкция, которая объясняет, как собрать наш контейнер.
На сайте docker есть подробное описание всех команд поддерживаемых DockerFile

Чтобы свободно пользоваться общей папкой в VM Ubuntu надо добавить себя в соответсвующую группу этой папки. Ее владельца (если хост ОС это винда) поменять не получиться.

Для того, чтобы Docker увидел Dockerfile файл должен называться именно "Dockerfile" соблюдая регистр букв.
	
При создании БД надо прописать директорию для хранения данных таблиц

Для выполнения команды при запуске контейнера в Dockerfile пишем команду ENTRYPOINT (еще можно использовать CMD) и прописываем команду

Для передачи клиенту mariadb скрипта SQL для выполнения в конце команды пишем: < "наш/файл/для/выполнения.sql" (< указывает передать содержимое файла команде)

Пароли нельзя загружать на публичные репозиториию. Чтобы этого избежать пароли хранятся в отдельном файле, который не загружают в удаленный реп., а их передача частям приложения идет при помощи переменных окружения и других методов
